# database/

PostgreSQL schema and seed data for the platform. **Provided by the academy.**

```
database/
├── schema/
│   └── 001_init.sql          # tables: profile, skills, certifications, projects
└── seed/
    ├── seed.sql.template     # SOURCE — edit customization.json, not this
    └── seed.sql              # GENERATED by `make customize` (your real data)
```

## How it works locally

`docker-compose.yml` mounts `schema/` and `seed/` into the postgres container's
init directory. The **first time** the database volume is created, postgres runs
every `*.sql` file in alphabetical order:

1. `schema/001_init.sql` creates the tables.
2. `seed/seed.sql` inserts your data.

> `seed.sql.template` ends in `.template`, **not** `.sql`, so postgres ignores
> it. Only the generated `seed.sql` is applied. Run `make customize` to generate
> it before `make dev`.

### Re-applying after changes

The init scripts only run on a fresh volume. To re-apply without wiping data:

```bash
make seed        # runs scripts/seed-db.sh against the running container
```

To start completely fresh (drops the volume):

```bash
make clean && make customize && make dev
```

## Your customization

The eight identity values come from `customization.json`. The skills,
certifications, and projects in `seed.sql` are **sample content** — once
generated, `seed.sql` is yours to personalize. Re-running `make customize`
regenerates it from the template, so make lasting content edits to the template
or keep a copy.

## What YOU will do later (Milestone 3)

- Provision **RDS PostgreSQL** with Terraform.
- Store the DB credentials in **AWS Secrets Manager** (never in plaintext).
- Run `schema/001_init.sql` and your `seed.sql` against RDS.
- Point the backend at RDS via the connection details from Secrets Manager.

See [docs/04-terraform-guide.md](../docs/04-terraform-guide.md).

## Tables

| Table | Purpose |
|---|---|
| `profile` | Single-row personal profile (name, title, bio, links). |
| `skills` | Skill list with category + level (1–5). |
| `certifications` | Certs with status `earned` / `in-progress` / `planned`. |
| `projects` | Portfolio projects with tech tags and links. |
